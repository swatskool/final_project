Project Technical Details

We scraped game information from the Roblox website as our starting point. Roblox has multiple categories of games to select from, with approximately 60 games for each category. We limited our scope to include 10 games from each of the Most Popular, Most Engaging, Up-and-Coming, and Top Earning games categories.


To create our Roblox site, we used JavaScript, PgAdmin (query), Python, Flask, and html. When viewing the Roblox site, the content continues to load after initial connection to the page. As a result, it was necessary to enable the Web Driver for the browser to allow faster loading speeds and to facilitate scraping of all content available.


We transformed the data to reflect the user rating as an integer to allow us to sort the games by user ratings. We also updated the game titles to ensure the YouTube query returned good results.


Using a static key word search and the game names returned from the web scrape, we queried the top views, like counts, comment counts, publish date and URL for the top 5 results returned from YouTube on "learn to play ".


We created drop-down menus to access our data and to make an easy selection for the user based on their preferences. First, the user will select from four categories. Then, the user selects their choice of game property choices (either game ratings or number of players). Next, the user will select their game of choice from the returned results (of 10 games per category). Once a game is selected, five of the top YouTube videos will appear for the user to view. Another drop-down menu will offer selection of how to view the YouTube videos (based on View Count, Comment Count, or Like Count). The URL for the current YouTube video is also displayed.
